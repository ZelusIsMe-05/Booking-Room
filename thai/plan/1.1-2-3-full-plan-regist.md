# Plan triển khai: Đăng ký + Xác thực OTP + Gửi lại OTP (1.1 + 1.2 + 1.3)

> Spec chi tiết từng API ở `1.1-register.md`, `1.2-verify-otp.md`, `1.3-resend-otp.md`.
> File này là **lộ trình code** theo phase. OTP: email-only, lưu Redis **plaintext**, TTL 300s.

## 0. Tổng quan 3 endpoint

| Endpoint | Method | Tóm tắt |
| --- | --- | --- |
| `/api/auth/register` | POST | Tạo user `INACTIVE` (+tenant +account_security) trong 1 transaction → set OTP Redis → gửi email |
| `/api/auth/verify-otp` | POST | So khớp OTP → `ACTIVE` + `DEL` key |
| `/api/auth/resend-otp` | POST | Cooldown 60s → ghi đè OTP mới → gửi email |

## Phase 1 — Đăng ký (`1.1`)

**`models/User.js`** — thêm:
- `registerSchema` (Zod): `fullName`, `username` (regex `^[a-z][a-z0-9_.]{2,49}$`, lowercase), `email`, `phoneNumber` (10 số), `password` (policy mạnh), `confirmPassword` (refine = password).
- `toRegisterResponse(user)` DTO (userId, username, email, phoneNumber, status).

**`repositories/auth/auth.repository.js`** — thêm:
- `findExistingByEmailPhoneUsername({ email, phoneNumber, username })` → check trùng.
- `createTenantUser(trx, {...})` → insert `users` (status INACTIVE) + `tenants` + `account_security` trong **transaction** (dùng `db.transaction`).
- `getRoleIdByName('TENANT')`.

**`services/auth/auth.service.js`** — thêm `register(input)`:
1. check trùng → `AppError 409`.
2. `getRoleIdByName('TENANT')`.
3. `hashPassword`.
4. transaction: tạo user+tenant+account_security.
5. **sau commit**: `generateOtp` → `otpStore.setOtp` → `mailer.sendOtpEmail`.
6. trả DTO + `otpExpiresInSeconds`.

**`controllers/auth/auth.controller.js`**: `register` → `201` + `sendSuccess`.
**`routes/auth/auth.route.js`**: `POST /register` + `validate({ body: registerSchema })`.

## Phase 2 — Xác thực OTP (`1.2`)

**`models/User.js`**: `verifyOtpSchema` (`email`, `otp` 6 số).
**Repository**: `findInactiveUserByEmail(email)`, `activateUser(userId)` (set ACTIVE + verified_at + updated_at).
**Service** `verifyOtp({ email, otp })`:
1. tìm user INACTIVE → không có ⇒ `404`.
2. `otpStore.verifyOtp` → `EXPIRED` ⇒ `410`; `attempts >= max` ⇒ `429`; `WRONG` ⇒ `incrAttempts`, `400` + `remainingAttempts`.
3. đúng ⇒ `deleteOtp` **trước** rồi `activateUser` (chống double-submit) ⇒ `200`.

**Controller/route**: `POST /verify-otp` + `validate`.

---

## Phase 3 — Gửi lại OTP (`1.3`)

**`models/User.js`**: `resendOtpSchema` (`email`, `purpose` = `REGISTRATION`).
**Service** `resendOtp({ email })`:
1. user INACTIVE? không ⇒ `404`; đã ACTIVE ⇒ `409`.
2. `isOnCooldown` ⇒ `429`.
3. `generateOtp` → `setOtp` (ghi đè) → `setResendCooldown` → `sendOtpEmail` ⇒ `200`.

**Controller/route**: `POST /resend-otp` + `validate`.

---

## Hằng số / chuẩn dùng lại
- `OTP_PURPOSE.REGISTRATION = 'REGISTRATION'` (đặt trong `config/authConstants.js`).
- Mọi lỗi nghiệp vụ ném `AppError(code, message, status, data?)`; response qua `sendSuccess`/`sendError`.
- `identifier` = email normalize (trim + lowercase) ở cả 3 flow.

## Thứ tự làm đề xuất
`Phase 0` → `Phase 1` (test register tạo user + OTP vào Redis) → `Phase 2` (test activate) → `Phase 3` (test resend + cooldown).

## Checklist test (sau mỗi phase)
- [ ] Register: user `INACTIVE`, key `otp:REGISTRATION:{email}` tồn tại (TTL ~300), email/console có OTP, trùng email/username → 409, validate sai → 400.
- [ ] Verify: OTP đúng → ACTIVE + key bị xóa; sai → remainingAttempts giảm; quá 3 lần → 429; hết hạn → 410.
- [ ] Resend: ghi đè OTP mới (attempts về 0), gọi lại trong 60s → 429.
- [ ] Không endpoint nào lộ OTP thô / password hash.
