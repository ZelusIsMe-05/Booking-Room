# 1.11. Tích hợp ảnh CCCD landlord lên AWS S3

Sau khi merge `origin/develop` (đã có S3 cho ảnh phòng), chuyển lưu trữ ảnh CCCD từ
**local disk** (`uploads/landlords/{id}/1.jpg|2.jpg`) sang **AWS S3**, dùng lại
`backend/utils/s3Helper.js` có sẵn. Mục tiêu: thống nhất hạ tầng lưu trữ với ảnh phòng,
**đồng thời giữ tính riêng tư của CCCD** (không public như ảnh phòng).

## Hiện trạng sau merge

- `utils/s3Helper.js`: `uploadToS3(buffer, key, mime) → full URL`, `deleteFromS3(key)`,
  `generateS3Key(type, entityId, originalName, index)`, `extractS3KeyFromUrl(url)`,
  `RESOURCE_TYPES = { AVATAR, ROOM, REPORT, SYSTEM }` — **chưa có loại cho CCCD**.
- `services/host/roomService.js`: upload ảnh phòng bằng `uploadToS3`, **lưu full URL** vào
  `room_images.image_url`. Có `assertVerifiedHost` (chỉ kiểm tra landlord có `id_card_*_url` khác null).
- `tools/uploadLocalImagesToS3.js`: script 1 lần migrate ảnh phòng `/uploads/%` → S3.
- Env (theo s3Helper): `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_S3_BUCKET_NAME`.
- `multer` dùng memoryStorage (`file.buffer`) → tương thích `uploadToS3`, **không cần đổi**.
- Phần landlord (của ta) đang dùng `services/storage/idCardStorage.js` (driver local) với
  `save / remove / getStream` và lưu **key tương đối** vào DB.

## Khác biệt cần dung hòa (quyết định thiết kế)

| Vấn đề | Ảnh phòng (develop) | Ảnh CCCD (đề xuất) |
| --- | --- | --- |
| Quyền truy cập | Public (hiển thị cho mọi người) | **Private** (chỉ Admin) — CCCD nhạy cảm |
| Giá trị lưu DB | Full S3 URL | **S3 key** (vd `landlords/{id}/1.jpg`) — vì object private, URL trực tiếp không xem được |
| Cách xem | Dùng URL trực tiếp | **Presigned URL** ngắn hạn (hoặc proxy stream qua backend) |
| Đặt tên key | `rooms/{id}/{ts}{idx}{ext}` | `landlords/{id}/1{ext}`, `2{ext}` (cố định → ghi đè khi đăng ký lại) |

> Lý do lưu **key** thay vì full URL cho CCCD: object để **private**, nên URL public
> `https://bucket.s3.../key` không mở trực tiếp được; lưu key gọn hơn và để backend tự ký
> presigned URL khi Admin cần xem. (`extractS3KeyFromUrl` vẫn dùng được nếu sau muốn đổi.)

## Phương án triển khai

**Giữ seam `idCardStorage`** làm lớp trừu tượng, chỉ **thay ruột** từ `fs` sang `s3Helper`.
Nhờ vậy `auth.service.registerLandlord` và controller admin **gần như không đổi** (vẫn gọi
`idCardStorage.save/remove`, và một hàm lấy URL cho admin). Đây là điểm lợi của thiết kế ở 1.10.

### 1. Bổ sung loại CCCD cho `s3Helper.js`

```js
const RESOURCE_TYPES = { AVATAR:'avatars', ROOM:'rooms', REPORT:'reports', SYSTEM:'system', ID_CARD:'landlords' };

// trong generateS3Key, thêm case (tên cố định theo index để ghi đè, không timestamp):
case RESOURCE_TYPES.ID_CARD:
  return `landlords/${entityId}/${index}${ext}`; // index = 1 (front) | 2 (back)
```

### 2. Đổi ruột `services/storage/idCardStorage.js` sang S3

- `save({ landlordId, frontFile, backFile })`:
  - `frontKey = generateS3Key(ID_CARD, landlordId, frontFile.originalname, 1)`
  - `await uploadToS3(frontFile.buffer, frontKey, frontFile.mimetype)` (tương tự back, index 2)
  - trả `{ frontKey, backKey }` (vẫn là **key**, lưu vào DB như cũ).
- `remove(landlordId)`: `deleteFromS3('landlords/{id}/1...')` + `2...`. Vì tên có `ext` động,
  lưu lại 2 key đã upload để xóa chính xác (hoặc xóa theo prefix nếu dùng ListObjects — đơn giản
  nhất: nhận `keys` cần xóa). Đề xuất: `remove(keys[])` nhận mảng key đã lưu DB.
- Thay `getStream(key)` (đọc fs) bằng `getViewUrl(key)`:
  - Dùng **presigned GET** (`@aws-sdk/s3-request-presigner` + `GetObjectCommand`), TTL ~5 phút.
  - Cần thêm package: `npm i @aws-sdk/s3-request-presigner`.
  - (Fallback nếu không muốn thêm package: `getObjectStream(key)` qua `GetObjectCommand` rồi pipe
    qua backend — giữ object private mà không cần presigner.)

### 3. `auth.service.registerLandlord` — gần như giữ nguyên

- Vẫn: sinh `user_id` ở app → `idCardStorage.save(...)` (giờ upload S3) **trước** transaction →
  `createUserWithRole(... idCardFrontUrl=frontKey ...)` → rollback `idCardStorage.remove(...)` nếu lỗi.
- Upload S3 là gọi mạng → đặt **ngoài** transaction (đã đúng từ 1.10, nay càng cần thiết).

### 4. Admin xem ảnh — đổi endpoint stream sang presigned

- `userService.getLandlordIdCardKey(userId, side)` giữ nguyên (trả key).
- Controller `getLandlordIdCard`:
  - Cách A (khuyến nghị): trả JSON `{ url: <presigned> }` (hoặc `302 redirect` tới presigned URL)
    thay vì stream file. FE mở URL trực tiếp.
  - Cách B: giữ stream qua backend bằng `GetObjectCommand` (proxy), không lộ URL.
- Bỏ phần chặn `/uploads/landlords` trong `app.js` nếu **không còn** lưu local (S3 đảm nhiệm).
  Nếu giữ dual-mode (xem mục 6) thì vẫn giữ chặn.

### 5. Migration dữ liệu cũ (nếu có landlord đã lưu local)

- Thêm `tools/uploadLocalIdCardsToS3.js` (theo mẫu `uploadLocalImagesToS3.js`):
  - Quét `landlords` có `id_card_front_url`/`id_card_back_url` dạng `landlords/%` cục bộ
    (hoặc `/uploads/landlords/%`), đọc file đĩa, `uploadToS3`, cập nhật cột bằng key S3.
- Seed cũ dùng URL giả `https://cdn.booking.local/...` → bỏ qua (không phải file thật).

### 6. (Tùy chọn) Driver toggle local|s3

- Để dev không có AWS vẫn chạy: `idCardStorage` đọc `STORAGE_DRIVER` (env), `s3` → dùng s3Helper,
  `local` → giữ code fs cũ. Mặc định `s3` (đồng bộ hướng team đã làm cho ảnh phòng).
- Nếu team đã quyết S3-only (như ảnh phòng), có thể bỏ qua mục này cho gọn.

### 7. Env + dependency

- Đã có (từ develop): `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_S3_BUCKET_NAME`.
- Thêm (cho presigned): `@aws-sdk/s3-request-presigner`. (Tùy chọn) `ID_CARD_URL_TTL_SECONDS=300`.
- **Bảo mật bucket**: prefix `landlords/` phải **private** (chặn public-read). Nếu bucket đang
  public-read toàn bộ cho ảnh phòng → tách policy theo prefix, hoặc dùng 2 bucket.

## Phạm vi sửa

| File | Thay đổi |
| --- | --- |
| `utils/s3Helper.js` | Thêm `RESOURCE_TYPES.ID_CARD` + case trong `generateS3Key` |
| `services/storage/idCardStorage.js` | Đổi ruột sang `uploadToS3/deleteFromS3`; `getStream`→`getViewUrl` (presigned) hoặc `getObjectStream` |
| `services/auth/auth.service.js` | Gần như giữ nguyên (đã gọi qua idCardStorage) |
| `controllers/admin/userController.js` | `getLandlordIdCard` trả presigned URL / proxy stream S3 |
| `app.js` | Bỏ/giữ chặn `/uploads/landlords` tùy có còn local hay không |
| `tools/uploadLocalIdCardsToS3.js` (mới) | Migrate ảnh CCCD local → S3 |
| `package.json` | (Nếu dùng presigned) thêm `@aws-sdk/s3-request-presigner` |

## Kiểm thử

| Case | Kỳ vọng |
| --- | --- |
| Đăng ký landlord | Ảnh lên S3 tại `landlords/{id}/1.*`,`2.*`; DB lưu key; không còn ghi `uploads/landlords` |
| DB transaction lỗi | `deleteFromS3` dọn object vừa upload |
| Admin xem ảnh | Nhận presigned URL mở được trong TTL; hết hạn → 403/expired |
| Truy cập object S3 không ký | Bị từ chối (private) |
| Người không phải Admin gọi endpoint | `403` |
| Migrate landlord cũ (nếu có) | Cột id_card cập nhật key S3, ảnh mở được |

## Ghi chú / rủi ro

- CCCD **bắt buộc private** trên S3 — khác ảnh phòng (public). Phải kiểm tra policy bucket trước khi chạy thật.
- `uploadToS3` hiện không set ACL (dựa policy bucket). Đảm bảo prefix `landlords/` không bị public.
- Giữ `idCardStorage` làm seam giúp đổi local↔S3 và viết test dễ; không gọi `s3Helper` rải rác trong service.
- `assertVerifiedHost` (roomService, từ develop) chỉ kiểm tra có `id_card_*_url` — vẫn đúng vì ta luôn
  lưu key thật. Tầng chặn duyệt là `requireApprovedLandlord` (route host) chạy trước.
