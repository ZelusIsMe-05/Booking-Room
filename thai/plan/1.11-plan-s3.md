# 1.11. Chuyển upload file từ server local sang AWS S3

## Mục tiêu

Chuyển các luồng upload ảnh đang phụ thuộc vào thư mục `backend/uploads` sang AWS S3 để backend không lưu blob/image trên ổ đĩa server. Trọng tâm hiện tại là ảnh CCCD của host/landlord, vì ảnh phòng và avatar đã dùng `backend/utils/s3Helper.js`.

## Căn cứ tài liệu và code hiện tại

- `docs/04-architecture.md` xác định AWS S3 là external file storage, giảm tải việc lưu blob/image trực tiếp trên Node.js.
- `docs/aws-s3-integration.md` yêu cầu upload buffer từ Multer memory storage lên S3 và lưu URL S3 vào database.
- `backend/package.json` đã có `@aws-sdk/client-s3`.
- `backend/utils/s3Helper.js` đã hỗ trợ `uploadToS3`, `deleteFromS3`, `extractS3KeyFromUrl` và đang được ảnh phòng/avatar dùng.
- `backend/config/multer.js` dùng `memoryStorage`, nên file nằm ở `file.buffer`, phù hợp upload trực tiếp lên S3.
- Điểm còn dùng local runtime là `backend/services/storage/idCardStorage.js`: ghi ảnh CCCD vào `backend/uploads/landlords/{landlordId}`.

## Phạm vi thay đổi

| File | Thay đổi |
| --- | --- |
| `backend/utils/s3Helper.js` | Load `backend/.env` ổn định, chỉ cấu hình credentials khi có đủ key; thêm `RESOURCE_TYPES.ID_CARD`; sinh key S3 cố định `landlords/{id}/1.jpg` và `landlords/{id}/2.jpg`. |
| `backend/services/storage/idCardStorage.js` | Bỏ ghi file local bằng `fs`; `save()` upload 2 ảnh CCCD lên S3 và trả full S3 URL để lưu DB; `remove()` xóa 2 object S3. |
| `backend/controllers/admin/userController.js` | Admin xem ảnh CCCD bằng redirect tới S3 URL. Nếu gặp dữ liệu legacy local key thì trả `409 LEGACY_ID_CARD_STORAGE` để yêu cầu migrate. |
| `thai/plan/1.11-plan-s3.md` | Ghi lại plan và quyết định triển khai S3. |

## Luồng mới cho `POST /api/profile/host-verification`

1. Client gửi `multipart/form-data`.
2. Field file bắt buộc:
   - `id_card_front`
   - `id_card_back`
3. `uploadIdCards` đọc file bằng Multer memory storage, không ghi disk.
4. Controller kiểm tra đủ 2 file.
5. Service gọi `idCardStorage.save()`.
6. `idCardStorage.save()` upload:
   - `id_card_front` -> S3 key `landlords/{landlordId}/1.jpg`
   - `id_card_back` -> S3 key `landlords/{landlordId}/2.jpg`
7. Repository cập nhật `landlords.id_card_front_url`, `landlords.id_card_back_url`.
8. Response `200`, message `host verification submitted`.

## Quyết định thiết kế

- Lưu full S3 URL trong DB để đồng bộ với `room_images.image_url` và `users.avatar_url`.
- Dùng key cố định cho CCCD theo landlord (`1.jpg`, `2.jpg`) để nộp lại sẽ ghi đè object cũ, tránh sinh rác theo timestamp.
- Không dùng `backend/uploads/landlords` cho upload mới.
- Tạm giữ `app.use('/uploads', express.static(...))` cho dữ liệu legacy khác nếu còn, nhưng ảnh CCCD mới không phụ thuộc route này.

## Việc còn cần làm nếu môi trường có dữ liệu cũ

Nếu DB đã có landlord lưu key local kiểu `landlords/{id}/1.jpg` hoặc `/uploads/landlords/...`, cần thêm/chạy script migration:

1. Quét bảng `landlords` có `id_card_front_url` / `id_card_back_url` không phải URL `http(s)`.
2. Đọc file từ `backend/uploads/landlords/...`.
3. Upload lên S3 bằng `uploadToS3`.
4. Cập nhật 2 cột trong `landlords` thành full S3 URL.
5. Sau khi xác nhận dữ liệu đã lên S3, có thể xóa thư mục local cũ.

## Checklist kiểm thử

| Case | Kỳ vọng |
| --- | --- |
| Host nộp đủ 2 ảnh | Object xuất hiện trên S3, DB lưu full S3 URL, response `200`. |
| Thiếu `id_card_front` hoặc `id_card_back` | `400 ID_CARD_REQUIRED`. |
| File không phải ảnh hoặc quá 5MB | `400 INVALID_UPLOAD`. |
| Landlord đã `APPROVED` nộp lại | `409 ALREADY_APPROVED`. |
| Landlord `REJECTED` nộp lại | Ảnh mới ghi đè S3, trạng thái về `PENDING`, xóa `rejection_reason`. |
| Admin xem ảnh CCCD mới | `302` redirect tới S3 URL. |
| Admin xem ảnh legacy local chưa migrate | `409 LEGACY_ID_CARD_STORAGE`. |

## Giả định

- `backend/.env` có `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_S3_BUCKET_NAME`.
- Bucket/prefix S3 cho ảnh cần đọc được theo chiến lược hiện tại của nhóm. Nếu CCCD phải private trước production, cần bổ sung presigned URL hoặc proxy stream qua backend thay vì redirect URL public.
