# 1.2. Xác thực OTP đăng ký

## Mục tiêu

Xây dựng API xác thực OTP cho tài khoản mới đăng ký. Nếu OTP hợp lệ, tài khoản chuyển từ `INACTIVE` sang `ACTIVE`.

## Luồng hoạt động

1. Client gửi email hoặc số điện thoại kèm OTP.
2. Backend tìm user đang `INACTIVE`.
3. Backend tìm OTP registration mới nhất, chưa consumed.
4. Backend kiểm tra OTP còn hạn.
5. Backend kiểm tra số lần nhập sai chưa quá 3.
6. Backend so sánh OTP với `otp_hash`.
7. Nếu OTP sai, tăng `attempt_count`.
8. Nếu OTP đúng, set `consumed_at` và cập nhật user `status = ACTIVE`.
9. Backend trả response xác thực thành công.

## API

| Thuộc tính | Giá trị |
| --- | --- |
| Method | `POST` |
| Path | `/api/auth/verify-otp` |
| Auth required | Không |
| Content-Type | `application/json` |

## Header

```http
Content-Type: application/json
Accept: application/json
```

## Request body

```json
{
  "email": "vana@example.com",
  "otp": "123456"
}
```

Hoặc:

```json
{
  "phoneNumber": "0912345678",
  "otp": "123456"
}
```

## Request schema

| Field | Type | Required | Rule |
| --- | --- | --- | --- |
| `email` | string | Có nếu không có `phoneNumber` | Đúng định dạng email |
| `phoneNumber` | string | Có nếu không có `email` | 10 số |
| `otp` | string | Có | 6 chữ số |

## HTTP code

| Code | Trường hợp |
| --- | --- |
| `200` | OTP hợp lệ, tài khoản được kích hoạt |
| `400` | Request thiếu dữ liệu hoặc OTP sai |
| `404` | Không tìm thấy tài khoản/OTP phù hợp |
| `410` | OTP hết hạn |
| `429` | OTP đã nhập sai quá 3 lần |
| `500` | Lỗi server |

## Response thành công

```json
{
  "success": true,
  "message": "Xác thực OTP thành công. Tài khoản đã được kích hoạt.",
  "data": {
    "userId": "9d971ee5-7e97-4d6c-b70b-d9a7a91b5a00",
    "status": "ACTIVE"
  }
}
```

## Response OTP sai

```json
{
  "success": false,
  "message": "Mã OTP không chính xác.",
  "data": {
    "remainingAttempts": 2
  }
}
```

## Response OTP hết hạn

```json
{
  "success": false,
  "message": "Mã OTP đã hết hạn. Vui lòng yêu cầu gửi lại mã mới."
}
```

## Response quá số lần nhập sai

```json
{
  "success": false,
  "message": "Mã OTP đã bị khóa do nhập sai quá số lần cho phép. Vui lòng yêu cầu gửi lại mã mới."
}
```

## Response schema

| Field | Type | Ghi chú |
| --- | --- | --- |
| `success` | boolean | Trạng thái request |
| `message` | string | Thông báo tổng quát |
| `data.userId` | uuid | ID user được kích hoạt |
| `data.status` | string | `ACTIVE` sau khi OTP đúng |
| `data.remainingAttempts` | number | Chỉ trả khi OTP sai |

## Bảng liên quan

### `users`

Đọc và cập nhật:

- Tìm theo `email` hoặc `phone_number`.
- Chỉ xử lý user `status = INACTIVE`.
- Cập nhật `status = ACTIVE` khi OTP đúng.

### `otp_verifications`

Đọc và cập nhật:

- Tìm OTP theo `user_id` và `purpose = REGISTRATION`.
- Điều kiện `consumed_at IS NULL`.
- Kiểm tra `expires_at`.
- Tăng `attempt_count` nếu OTP sai.
- Set `consumed_at = now` nếu OTP đúng.

## Điều kiện cần để thực hiện

1. User đã đăng ký và đang `INACTIVE`.
2. Có OTP registration chưa consumed.
3. OTP chưa hết hạn 5 phút.
4. `attempt_count < 3`.

## Kết quả mong đợi

1. OTP đúng kích hoạt tài khoản.
2. OTP sai không kích hoạt tài khoản.
3. OTP hết hạn bị từ chối.
4. OTP nhập sai quá 3 lần bị từ chối và yêu cầu gửi lại.
5. OTP đã consumed không dùng lại được.

## Ghi chú triển khai

- Nên xử lý verify OTP trong transaction để tránh double-submit kích hoạt nhiều lần.
- Không trả về `otp_hash`.
- Không cấp JWT tự động ở bước này trừ khi sản phẩm chốt auto-login.
