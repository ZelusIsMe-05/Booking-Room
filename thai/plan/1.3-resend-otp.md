# 1.3. Gửi lại OTP

## Mục tiêu

Xây dựng API gửi lại OTP đăng ký cho user chưa kích hoạt tài khoản.

## Luồng hoạt động

1. Client gửi email hoặc số điện thoại.
2. Backend tìm user đang `INACTIVE`.
3. Backend vô hiệu OTP registration cũ chưa consumed nếu cần.
4. Backend sinh OTP mới.
5. Backend hash OTP và lưu vào `otp_verifications`.
6. Backend gửi OTP mới qua email.
7. Backend trả response thành công.

## API

| Thuộc tính | Giá trị |
| --- | --- |
| Method | `POST` |
| Path | `/api/auth/resend-otp` |
| Auth required | Không |
| Content-Type | `application/json` |

## Header

```http
Content-Type: application/json
Accept: application/json
```

## Request body

```json
{
  "email": "vana@example.com",
  "purpose": "REGISTRATION"
}
```

Hoặc:

```json
{
  "phoneNumber": "0912345678",
  "purpose": "REGISTRATION"
}
```

## Request schema

| Field | Type | Required | Rule |
| --- | --- | --- | --- |
| `email` | string | Có nếu không có `phoneNumber` | Đúng định dạng email |
| `phoneNumber` | string | Có nếu không có `email` | 10 số |
| `purpose` | string | Có | MVP chỉ nhận `REGISTRATION` |

## HTTP code

| Code | Trường hợp |
| --- | --- |
| `200` | OTP mới đã được gửi |
| `400` | Request không hợp lệ |
| `404` | Không tìm thấy tài khoản đang chờ xác thực |
| `409` | Tài khoản đã active, không cần gửi OTP đăng ký |
| `429` | Gửi lại quá thường xuyên nếu có rate limit |
| `500` | Lỗi server hoặc lỗi gửi email |

## Response thành công

```json
{
  "success": true,
  "message": "Mã OTP mới đã được gửi.",
  "data": {
    "otpExpiresInSeconds": 300
  }
}
```

## Response tài khoản đã active

```json
{
  "success": false,
  "message": "Tài khoản đã được kích hoạt."
}
```

## Response schema

| Field | Type | Ghi chú |
| --- | --- | --- |
| `success` | boolean | Trạng thái request |
| `message` | string | Thông báo tổng quát |
| `data.otpExpiresInSeconds` | number | `300` giây |

## Bảng liên quan

### `users`

Đọc:

- Tìm theo `email` hoặc `phone_number`.
- Chỉ cho resend nếu `status = INACTIVE`.

### `otp_verifications`

Ghi/cập nhật:

- Có thể set `consumed_at = now` cho OTP cũ chưa consumed.
- Insert OTP mới với `purpose = REGISTRATION`, `attempt_count = 0`, `expires_at = now + 5 minutes`.

## Điều kiện cần để thực hiện

1. User tồn tại.
2. User đang `INACTIVE`.
3. Có cấu hình gửi email OTP.
4. Nếu áp dụng rate limit, request chưa vượt giới hạn.

## Kết quả mong đợi

1. OTP cũ không còn dùng được nếu đã bị vô hiệu.
2. OTP mới có hạn 5 phút.
3. OTP mới được gửi đến email đã đăng ký.
4. Không trả OTP raw trong response.

## Ghi chú triển khai

- Nên có rate limit resend, ví dụ mỗi 60 giây chỉ gửi lại 1 lần.
- MVP có thể chưa thêm bảng rate limit riêng; có thể dựa vào `created_at` OTP mới nhất.
