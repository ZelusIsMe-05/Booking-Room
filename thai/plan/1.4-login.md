# 1.4. Đăng nhập

## Mục tiêu

Xây dựng API đăng nhập bằng email/số điện thoại/username và mật khẩu. Nếu thông tin hợp lệ, backend cấp một cặp JWT gồm **access token** (ngắn hạn) và **refresh token** (dài hạn) kèm thông tin user.

## Luồng hoạt động

1. Client gửi `identifier` và `password`.
2. Backend tìm user theo email, số điện thoại hoặc username.
3. Backend ghi audit failed nếu không tìm thấy user.
4. Backend kiểm tra tài khoản có đang bị khóa không.
5. Backend kiểm tra trạng thái user.
6. Backend so sánh mật khẩu bằng bcrypt.
7. Nếu sai mật khẩu, tăng số lần đăng nhập sai.
8. Nếu sai quá 5 lần liên tiếp, khóa tài khoản 10 phút.
9. Nếu đúng mật khẩu, reset số lần sai và cập nhật `last_login_at`.
10. Backend ghi audit success.
11. Backend tạo cặp access token + refresh token và trả `{ tokenType, accessToken, accessExpiresIn, refreshToken, refreshExpiresIn, user }`.

## API

| Thuộc tính | Giá trị |
| --- | --- |
| Method | `POST` |
| Path | `/api/auth/login` |
| Auth required | Không |
| Content-Type | `application/json` |

## Header

```http
Content-Type: application/json
Accept: application/json
User-Agent: Mozilla/5.0
```

Backend cần đọc IP từ request:

- `req.ip`
- hoặc `X-Forwarded-For` nếu chạy sau proxy tin cậy

## Request body

```json
{
  "identifier": "vana@example.com",
  "password": "Password@123"
}
```

## Request schema

| Field | Type | Required | Rule |
| --- | --- | --- | --- |
| `identifier` | string | Có | Email, số điện thoại hoặc username |
| `password` | string | Có | Không rỗng |

## HTTP code

| Code | Trường hợp |
| --- | --- |
| `200` | Đăng nhập thành công |
| `400` | Thiếu identifier/password |
| `401` | Sai tài khoản hoặc mật khẩu |
| `403` | Tài khoản chưa active hoặc bị banned |
| `423` | Tài khoản đang bị khóa tạm thời |
| `500` | Lỗi server |

## Response thành công

```json
{
  "success": true,
  "message": "Đăng nhập thành công.",
  "data": {
    "tokenType": "Bearer",
    "accessToken": "jwt-access-token",
    "accessExpiresIn": "15m",
    "refreshToken": "jwt-refresh-token",
    "refreshExpiresIn": "7d",
    "user": {
      "userId": "9d971ee5-7e97-4d6c-b70b-d9a7a91b5a00",
      "fullName": "Nguyen Van A",
      "email": "vana@example.com",
      "phoneNumber": "0912345678",
      "username": "vana",
      "role": "TENANT",
      "status": "ACTIVE"
    }
  }
}
```

## Response sai thông tin

```json
{
  "success": false,
  "message": "Tài khoản hoặc mật khẩu không chính xác."
}
```

## Response tài khoản bị khóa

```json
{
  "success": false,
  "message": "Tài khoản đang bị khóa tạm thời. Vui lòng thử lại sau.",
  "data": {
    "lockedUntil": "2026-06-13T05:30:00.000Z"
  }
}
```

## Response tài khoản chưa active

```json
{
  "success": false,
  "message": "Tài khoản chưa được xác thực OTP."
}
```

## Response schema

| Field | Type | Ghi chú |
| --- | --- | --- |
| `success` | boolean | Trạng thái request |
| `message` | string | Thông báo tổng quát |
| `data.tokenType` | string | `Bearer` |
| `data.accessToken` | string | JWT access token (ngắn hạn) |
| `data.accessExpiresIn` | string | Theo `JWT_ACCESS_EXPIRES_IN` |
| `data.refreshToken` | string | JWT refresh token (dài hạn) |
| `data.refreshExpiresIn` | string | Theo `JWT_REFRESH_EXPIRES_IN` |
| `data.user` | object | User public data |
| `data.lockedUntil` | datetime | Chỉ có khi tài khoản bị khóa (trong nhánh lỗi `423`) |

## JWT payload

Access token mang định danh + ngữ cảnh phân quyền để các module khác rút UUID từ `sub`:

```json
{
  "sub": "9d971ee5-7e97-4d6c-b70b-d9a7a91b5a00",
  "role": "TENANT",
  "status": "ACTIVE",
  "type": "access"
}
```

Refresh token tối giản, chỉ dùng để cấp lại access token:

```json
{
  "sub": "9d971ee5-7e97-4d6c-b70b-d9a7a91b5a00",
  "type": "refresh"
}
```

Hai token ký bằng secret riêng (`JWT_ACCESS_SECRET`, `JWT_REFRESH_SECRET`). Access token chỉ gửi kèm ở các request tài nguyên; refresh token chỉ gửi tới `POST /api/auth/refresh`. Cơ chế hiện tại là **stateless, không rotation**: refresh token được dùng lại tới khi hết hạn.

## Bảng liên quan

### `users`

Đọc:

- Tìm theo `email`, `phone_number`, hoặc `username`.
- Lấy `password`, `status`, `role_id`.

### `roles`

Đọc:

- Join để lấy `role_name`.

### `account_security`

Đọc/cập nhật:

- `failed_login_attempts`
- `locked_until`
- `last_login_at`
- `updated_at`

### `login_audit_logs`

Ghi mới cho mọi lần login:

- `user_id`
- `login_identifier`
- `success`
- `failure_reason`
- `ip_address`
- `user_agent`
- `created_at`

## Điều kiện cần để thực hiện

1. User đã tồn tại.
2. User phải `ACTIVE` để đăng nhập thành công.
3. User không bị `BANNED`.
4. Tài khoản không đang bị khóa tạm thời.
5. `JWT_ACCESS_SECRET` và `JWT_REFRESH_SECRET` đã cấu hình.
6. `JWT_ACCESS_EXPIRES_IN` (default `15m`) và `JWT_REFRESH_EXPIRES_IN` (default `7d`) đã cấu hình.

## Kết quả mong đợi

1. Đăng nhập đúng trả về cặp access token + refresh token.
2. Đăng nhập sai không tiết lộ tài khoản có tồn tại hay không.
3. Sai quá 5 lần liên tiếp khóa tài khoản 10 phút.
4. Đăng nhập đúng reset số lần sai.
5. Mọi lần đăng nhập đều có audit log.
6. Response không trả password hash.

## Ghi chú triển khai

- Lỗi sai user và sai password đều dùng cùng message chung.
- Nên ghi audit log cả khi user không tồn tại, với `user_id = null`.
- Cần cẩn thận timezone khi tính `locked_until`.
