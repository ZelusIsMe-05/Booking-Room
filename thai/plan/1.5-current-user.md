# 1.5. Lấy thông tin user hiện tại

## Mục tiêu

Xây dựng API đọc thông tin user hiện tại dựa trên JWT hợp lệ.

## Luồng hoạt động

1. Client gửi request kèm `Authorization: Bearer <token>`.
2. Middleware verify JWT.
3. Backend lấy `userId` từ token.
4. Backend truy vấn user và role trong database.
5. Backend kiểm tra user còn `ACTIVE`.
6. Backend trả thông tin user public.

## API

| Thuộc tính | Giá trị |
| --- | --- |
| Method | `GET` |
| Path | `/api/auth/me` |
| Auth required | Có |
| Content-Type | Không bắt buộc |

## Header

```http
Accept: application/json
Authorization: Bearer <jwt-token>
```

## Request body

Không có.

## HTTP code

| Code | Trường hợp |
| --- | --- |
| `200` | Token hợp lệ và user tồn tại |
| `401` | Thiếu token, token sai hoặc token hết hạn |
| `403` | User không còn active hoặc bị banned |
| `404` | User trong token không còn tồn tại |
| `500` | Lỗi server |

## Response thành công

```json
{
  "success": true,
  "message": "Lấy thông tin người dùng thành công.",
  "data": {
    "user": {
      "userId": "9d971ee5-7e97-4d6c-b70b-d9a7a91b5a00",
      "fullName": "Nguyen Van A",
      "email": "vana@example.com",
      "phoneNumber": "0912345678",
      "username": "vana",
      "avatarUrl": null,
      "role": "TENANT",
      "status": "ACTIVE"
    }
  }
}
```

## Response thiếu token

```json
{
  "success": false,
  "message": "Bạn cần đăng nhập để thực hiện thao tác này."
}
```

## Response token không hợp lệ

```json
{
  "success": false,
  "message": "Phiên đăng nhập không hợp lệ hoặc đã hết hạn."
}
```

## Response schema

| Field | Type | Ghi chú |
| --- | --- | --- |
| `success` | boolean | Trạng thái request |
| `message` | string | Thông báo tổng quát |
| `data.user.userId` | uuid | ID user |
| `data.user.fullName` | string | Họ tên |
| `data.user.email` | string | Email |
| `data.user.phoneNumber` | string/null | Số điện thoại |
| `data.user.username` | string | Username |
| `data.user.avatarUrl` | string/null | Ảnh đại diện |
| `data.user.role` | string | `TENANT`, `LANDLORD`, `ADMIN` |
| `data.user.status` | string | Trạng thái tài khoản |

## Bảng liên quan

### `users`

Đọc:

- `user_id`
- `full_name`
- `email`
- `phone_number`
- `username`
- `avatar_url`
- `status`
- `role_id`

### `roles`

Đọc:

- `role_name`

## Điều kiện cần để thực hiện

1. JWT middleware đã hoạt động.
2. Token được ký bằng đúng `JWT_SECRET`.
3. User trong token còn tồn tại.
4. User đang `ACTIVE`.

## Kết quả mong đợi

1. Token hợp lệ trả đúng user.
2. Token thiếu/sai/hết hạn bị từ chối.
3. User `BANNED` hoặc `INACTIVE` không được xem là phiên hợp lệ.
4. Response không trả password hash.
