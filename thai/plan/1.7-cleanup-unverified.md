# 1.7. Dọn dẹp đăng ký chưa xác thực OTP

## Mục tiêu

Xây dựng service task dọn dẹp hoặc vô hiệu hóa các bản ghi đăng ký chưa xác thực OTP sau 15 phút, nhằm hạn chế đăng ký rác.

## Luồng hoạt động

1. Task chạy định kỳ, ví dụ mỗi 5 phút.
2. Backend tìm các user `INACTIVE` được tạo quá 15 phút và chưa xác thực OTP.
3. Backend xác định user chưa có OTP `REGISTRATION` nào consumed.
4. Backend vô hiệu OTP còn mở.
5. Backend xóa user hoặc giữ user ở trạng thái `INACTIVE` theo quyết định triển khai.
6. Backend ghi log nếu cần.

## API

Không public API cho client.

Đề xuất triển khai dưới dạng internal service:

```text
authCleanupService.cleanupUnverifiedRegistrations()
```

Có thể gọi bằng:

- cron nội bộ khi server start
- script thủ công
- job scheduler của môi trường deploy

## Header

Không áp dụng.

## Request body

Không áp dụng.

## HTTP code

Không áp dụng vì đây không phải public API.

Nếu triển khai endpoint nội bộ sau này:

| Code | Trường hợp |
| --- | --- |
| `200` | Cleanup chạy thành công |
| `401` | Thiếu internal secret |
| `500` | Lỗi server |

## Response nội bộ đề xuất

```json
{
  "success": true,
  "data": {
    "deletedUsers": 3,
    "expiredOtps": 5
  }
}
```

## Bảng liên quan

### `users`

Đọc/xóa hoặc cập nhật:

- Điều kiện `status = INACTIVE`.
- Điều kiện thời gian đăng ký quá 15 phút.

Lưu ý: migration hiện tại chưa có `created_at` trong `users`. Nếu chưa có timestamp tạo user, cần bổ sung migration hoặc dùng timestamp từ `otp_verifications.created_at` để xác định tuổi đăng ký.

### `otp_verifications`

Đọc/cập nhật:

- `purpose = REGISTRATION`
- `consumed_at IS NULL`
- `created_at < now - 15 minutes`

### `tenants`

Nếu xóa user, bảng `tenants` sẽ bị xóa theo FK cascade nếu migration đã cấu hình `onDelete('CASCADE')`.

## Điều kiện cần để thực hiện

1. Có cách xác định thời điểm đăng ký hoặc thời điểm tạo OTP.
2. Có strategy rõ ràng:
   - xóa user chưa xác thực
   - hoặc giữ user `INACTIVE` và vô hiệu OTP
3. FK cascade không làm ảnh hưởng dữ liệu ngoài ý muốn.
4. Task không chạy đồng thời nhiều instance gây race condition.

## Kết quả mong đợi

1. Đăng ký chưa xác thực quá 15 phút không còn dùng OTP cũ.
2. Email/phone của đăng ký rác được giải phóng nếu chọn xóa user.
3. Không ảnh hưởng user đã `ACTIVE`.
4. Không xóa user có OTP đã consumed.

## Ghi chú triển khai

- Nếu muốn đúng yêu cầu "xóa hoặc vô hiệu hóa", MVP nên chọn xóa user `INACTIVE` chưa consumed OTP để giải phóng email/phone.
- Nếu chọn không xóa user, cần định nghĩa rõ user `INACTIVE` cũ có được resend OTP sau 15 phút hay không.
- Nên log số lượng bản ghi cleanup để dễ kiểm tra vận hành.
