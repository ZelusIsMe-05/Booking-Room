## Validation / auth behavior

- Missing token -> 401 from `requireAuth`.
- Non-admin token -> 403 from `authorizeRoles('ADMIN')`.
- Invalid token -> 401 from `requireAuth`.

## Manual verification commands

App load:

```bash
node -e "require('./app'); console.log('app loaded')"
```

Health:

```bash
node -e "const app=require('./app'); const server=app.listen(0, async()=>{ const port=server.address().port; try { const res=await fetch('http://127.0.0.1:'+port+'/health'); console.log(res.status, await res.text()); } finally { server.close(); } });"
```

DB health needs database network permission:

```bash
node -e "const app=require('./app'); const db=require('./config/db'); const server=app.listen(0, async()=>{ const port=server.address().port; try { const res=await fetch('http://127.0.0.1:'+port+'/health/db'); console.log(res.status, await res.text()); } finally { server.close(async()=>{ await db.destroy(); }); } });"
```

Admin smoke test needs database network permission:

```bash
node -e "const app=require('./app'); const db=require('./config/db'); const server=app.listen(0, async()=>{ const port=server.address().port; const base='http://127.0.0.1:'+port; try { const login=await fetch(base+'/api/auth/login',{method:'POST',headers:{'content-type':'application/json'},body:JSON.stringify({identifier:'admin@booking.local',password:'Password@123'})}); const loginBody=await login.json(); console.log('login', login.status, loginBody.success); const token=loginBody.data && loginBody.data.accessToken; for (const path of ['/api/admin/dashboard/overview','/api/admin/system-logs?limit=3']) { const res=await fetch(base+path,{headers:{authorization:'Bearer '+token}}); console.log(path, res.status, await res.text()); } } finally { server.close(async()=>{ await db.destroy(); }); } });"
```

## Known side effect

Every successful admin smoke login inserts one `AUTH_LOGIN_SUCCESS` row into `system_logs`.

## Admin user APIs

### GET `/api/admin/users`

Auth: ADMIN bearer token

Query:

| Field | Meaning |
| --- | --- |
| `page` | Page number, default 1 |
| `limit` | Page size, default 20, max 100 |
| `role` | `ADMIN`, `LANDLORD`, `TENANT` |
| `status` | `ACTIVE`, `INACTIVE`, `BANNED` |
| `keyword` | Search full name, email, phone, username |

### GET `/api/admin/users/:id`

Auth: ADMIN bearer token

Returns user detail, role, landlord/tenant presence, and account security summary.

### PATCH `/api/admin/users/:id/lock`

Auth: ADMIN bearer token

Body:

```json
{
  "reason": "Manual review"
}
```

Sets `users.status = BANNED`.

### PATCH `/api/admin/users/:id/unlock`

Auth: ADMIN bearer token

Sets `users.status = ACTIVE` and clears account lock counters.

### PATCH `/api/admin/users/:id/role`

Auth: ADMIN bearer token

Body:

```json
{
  "role": "TENANT"
}
```

### POST `/api/admin/users/:id/password-reset`

Auth: ADMIN bearer token

Body:

```json
{
  "temporaryPassword": "Password@123"
}
```

Hashes the temporary password with bcrypt before saving.

Validation:

- Missing user on lock/detail/update/reset -> 404.
- Invalid role -> 400.
- Temporary password shorter than 8 chars -> 400.
- Non-admin token -> 403.

Database writes:

- `users` on lock/unlock/role/password reset.
- `account_security` on unlock/password reset.
- `tenants` when assigning `TENANT` to a user without a tenant row.
- `system_logs` on admin user actions.

Smoke test results:

```text
GET /api/admin/users?role=TENANT&limit=2 -> 200
GET /api/admin/users/:id -> 200
PATCH /api/admin/users/00000000-0000-0000-0000-000000000000/lock -> 404
PATCH /api/admin/users/:id/role { "role": "NOPE" } -> 400
PATCH /api/admin/users/:id/lock -> 200
PATCH /api/admin/users/:id/unlock -> 200
PATCH /api/admin/users/:id/role { "role": "TENANT" } -> 200
POST /api/admin/users/:id/password-reset -> 200
GET /api/admin/users with tenant token -> 403
```

Additional side effect:

Admin user smoke tests insert admin action rows into `system_logs`.
